  Architecture

  Zustand (useAppStore.settings.theme)  ← persisted to AsyncStorage
           ↓
  ThemeProvider (buildTheme(mode))       ← memoized Theme object
           ↓
  useTheme() / ThemedView / ThemedText   ← consume colors, spacing, type

  The Zustand store remains the single source of truth — ThemeToggle calls updateSettings({ theme }) which triggers a re-render of ThemeProvider, rebuilding the theme object. No duplicate AsyncStorage persistence needed since Zustand's persist
  middleware already handles it.
